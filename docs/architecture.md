## 3. 技術棧

| 元件 | 技術 | 用途 |
|------|------|------|
| 流程編排 | n8n | Scheduler、Webhook、Workflow |
| 本地 LLM | Ollama (llama3.2) | 案件評分、報價草稿、修改整理 |
| 動態爬蟲 | Playwright | 抓取動態頁面 |
| 靜態抽取 | trafilatura | 文字內容提取 |
| 人機互動 | Telegram Bot API | 通知與回覆 |
| 資料庫 | PostgreSQL 15 | 案件、專案、狀態、日誌 |
| 快取 | Redis 7 | Session、Queue、Rate Limit |
| Agent 治理 | Paperclip | 多 agent 狀態監控 |
| 開發執行 | Claude Code / Codex | 程式碼生成與修改 |
| 容器化 | Docker Compose | 一鍵啟動所有服務 |

## 4. Docker Compose 服務拓樸

```
docker-compose.yml
├── n8n          (port 5678)   — 流程編排 + webhook
├── ollama       (port 11434)  — 本地 LLM
├── postgres     (port 5432)   — 主資料庫
├── redis        (port 6379)   — 快取與 queue
└── telegram-bot (internal)    — Bot service
```

服務間通訊走 Docker 內部網路（`agency-net`），僅 n8n 對外暴露 5678 port。

## 6. 安全與合規

- **PRO360**：僅抓公開列表頁，禁止登入後操作
- **速率控制**：請求間隔 3–10 秒隨機，User-Agent 輪換
- **資料隔離**：所有 LLM 推理在本地完成
- **人工守門**：報價送出、客戶溝通必須人工確認
- **日誌稽核**：所有 agent 操作寫入 `agent_logs` 表
